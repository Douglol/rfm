<h1>Relatório técnico do projeto de RFM</h1>
<h2>Visão Geral do Projeto</h2>
<p>O objetivo foi compreender melhor os padrões de comportamento dos clientes, 
  segmentando-os de forma mais estratégica para apoiar decisões de marketing, 
  retenção e personalização.</p>

<h2>Componentes da Arquitetura</h2>

<h3>1. SQL e Python:</h3>
<ul>
  <li>Criação das tabelas no DuckDB.</li>
  <li>Criação das views.</li>
  <li>Criação das funções.</li>
  <li>Geração dos arquivos finais.</li>
</ul>

<h3>2. Power BI</h3>
<ul>
  <li>Criação do Dashboard.</li>
</ul>

<h2>Benefícios e Justificativas Técnicas</h2>
<ul>
  <li>Identificação de segmentos com comportamentos claros e acionáveis.</li>
  <li>Possibilidade de campanhas direcionadas com maior eficiência.</li>
  <li>Base sólida para estratégias de CRM e recomendações personalizadas.</li>
  <li>Foi um projeto que uniu estatística, modelagem e visão de negócio.</li>
</ul>
